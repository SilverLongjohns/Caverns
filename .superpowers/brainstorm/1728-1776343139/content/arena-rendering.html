How should the tactics arena render?

Your existing tile grid is ASCII rendered in a monospace pre block. Here are three directions for the combat arena.

A

Pure ASCII (same as exploration)

Reuse the exact same TileGridView renderer. Players are @, mobs are letters, walls are #. Movement range shown with highlight colors. Consistent with the rest of the game's CRT aesthetic. Fast to build.

################ #..............# #...@...#...G.# #......#.......# #.............# #~~......#....# #~~............# #..........S..# #...#..........# #..@....G....# ################ Movement range shown as highlighted tiles
B

ASCII with grid overlay

Same ASCII characters but rendered on a visible grid (subtle cell borders). Makes tile boundaries obvious for counting movement. Hover/click cells to move. Slightly more "game board" feel.

C

Tile-based with simple sprites

Ditch monospace text. Render a proper tile grid using small colored squares or simple emoji/unicode sprites in a CSS grid. More visual clarity for terrain types. Bigger departure from the CRT aesthetic.
